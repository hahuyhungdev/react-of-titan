# 🌐 Deployment

## Build

```bash
npm run build
```

This runs TypeScript check (`tsc -b`) followed by Vite production build. Output goes to `dist/`.

## Preview

```bash
npm run preview
```

Serves the production build locally for verification before deployment.

## CI/CD

A GitHub Actions workflow (`.github/workflows/ci.yml`) runs on every push:

1. Install dependencies
2. Format check (`npm run format:check`)
3. Lint (`npm run lint`)
4. Architecture boundary check (`npm run arch:check`)
5. Type-check (`npm run typecheck`)
6. Test (`npm test`)
7. Build (`npm run build`)

## Deploy Targets

The app is a static SPA — deploy to any CDN or static hosting:

| Platform     | Command                           |
| ------------ | --------------------------------- |
| Vercel       | `vercel deploy`                   |
| Netlify      | `netlify deploy`                  |
| CloudFront   | Upload `dist/` to S3 + CloudFront |
| GitHub Pages | Use `actions/deploy-pages`        |

## Environment Variables

Set environment variables on the hosting platform. Only `VITE_` prefixed variables are exposed to the client.

| Variable            | Description  |
| ------------------- | ------------ |
| `VITE_API_BASE_URL` | API base URL |

## Source Maps

Source maps are generated by default. For production, upload them to error tracking (Sentry) and exclude from public deployment if security is a concern.
